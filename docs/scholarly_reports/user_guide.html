Abstract

Unsplash Semantic API (USe) is a Web application based on the Semantic Web principles. This document aims to provide users with documentation on using the application, detailing its various features, and giving an overview of the application's endpoints that can be of service to the advanced users.

Introduction

USe stands for Unsplash Semantic API, and is a Web application following the Semantic Web principles.

The application aims to allow users to perform various tasks such as:

  • Visualize - images as the result of searching or using predefined collections. Individual images have associated properties and (meta-)informations, enriching the knowledge about a photograph.
  • Augment - both visually, using parameters like brightness, contrast, blur, etc., and semantically, with additional metadata about photo-related concepts, retrieved from external datasets like DBpedia1 and Wikidata2.
  • Filter and Search - on various criterias such as photo id, keywords, photo location country or city, photographer name, camera maker, etc. which provide users the ability to quickly find and explore photos that they may find of interest.

USe is a multi-device Web application, accessible to the users on both desktop and mobile platforms. The application also embeds structured data into the HTML using RDFa3, in order for the data to be parseable by Semantic Web tools. Moreover, all the data is accessible through both traditional REST APIs and SPARQL endpoints that give users full access to the data about nature-themed photos provided for free by the Unsplash Lite Dataset4 that is backing-up the application.

Web Pages of the Application

The Front-End application was developed using the Angular framework, with the addition of Angular Material Design components and additional styling with Bootstrap 5. The Front-End application aims to offer an attractive and easy to use interface for the users in order to access the functionality of the backend in an intuitive manner. Also, the application is organised in a set of pages with well defined functionality.

Search and Filter Photos Page

This page provides the users with various controls to search photos, for example by description or keywords, or to filter them by criterias like photographer, country, city, camera make, etc. The page also displays the search results in a grid-like manner and allows users to click any photo result, and go to their individual page.

Individual Photo Page

This page allows the users to view a particular photo with a larger resolution size, to view it's source Unsplash URL, it's description, associated keywords, the number of downloads and views, exif data like camera model and ISO, submission date, photo location country and city, the GPS coordinates, etc. This page is also augmented with data from external datastores like DBpedia about concepts like the photo location country, city, about the place, etc.

Search and Filter Collections Page

This page provides the users with various controls to search collections, for example, by title. The page also displays the results in a grid-like manner and allows users to click the collection results, which lead to their individual page.

Individual Collection Page

This page allows the users to view a particular collection. It displays the collection title and the photos that make up the collection in a grid-like manner. Users can then click a photo result, and go to its individual page to find out more details.

REST API Guide

The application also offers a REST API, and it's interface description follows the OpenAPI Specification v3.1.05. The APIs provides access through the HTTP(S) protocol to the data that is also backing-up the application.

Photos Endpoints

The information related to photos can be retrieved via a set of endpoints, and can be customized via query parameters. For example, photos can be searched or filtered via one or multiple criterias, in a case-insensitive manner for the string query parameters. Pagination support is offered via offset (default is 0) and limit (default is 15) query parameters.

The endpoints for photos are:

  • GET /photos Retrieves photos without searching or filtering.
  • GET /photos/{photoId} Retrieves a particular photo by id.
  • GET /photos?country={country} Retrieves photos filtered by country name.
  • GET /photos?city={city} Retrieves photos filtered by city name.
  • GET /photos?cameraMake={cameraMake} Retrieves photos filtered by camera maker.
  • GET /photos?photographerFirstName={name} Gets photos filtered by photographer first name.
  • GET /photos?photographerLastName={name} Gets photos filtered by photographer last name.

  • GET /photos/search?country={country} Retrieves photos searched by country name.
  • GET /photos/search?city={city} Retrieves photos searched by city name.
  • GET /photos/search?cameraMake={cameraMake} Retrieves photos searched by camera maker.
  • GET /photos/search?photographerFirstName={name} Retrieves photos searched by photographer first name.
  • GET /photos/search?photographerLastName={name} Retrieves photos searched by photographer last name.
  • GET /photos/search?keyword={keyword} Retrieves photos searched by keyword.

  • GET /additional-data/city/{city} Retrieves additional data about a city by name (from external data source - DBpedia).

Collections Endpoints

Collections can also be retrieved via a set of endpoints, and the queries can be customized via query parameters. For example, collections can be searched by title in a case-insensitive manner. Pagination support is offered via offset (default is 0) and limit (default is 15) query parameters.

The endpoints for collections are:

  • GET /collections Retrieves collections without filtering.
  • GET /collections/{collectionId} Retrieves a particular collection by id.
  • GET /collections?title={title} Retrieves collections filtered by title.
  • GET /collections/search?title={title} Retrieves collections searched by title.
